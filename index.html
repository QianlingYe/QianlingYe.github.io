List vs Dictionary

List: indexed by range of numbers; collection of values order matters
Dictionary: indexed by unique keys; lookup table with unique keys

An Example:

Intro to Python for Data Science

Python Types
  1. int, or integer: a number without a fractional part. eg:100
  2. float, or floating point: a number that has both an integer and fractional part, separated by a point. eg:1.01

Next to numerical data types, there are two other very common data types:

  1. str, or string: a type to represent text. You can use single or double quotes to build a string.
  2. bool, or boolean: a type to represent logical values. Can only be True or False.

To find out the type of a value or a variable that refers to that value, you can use the type() function.
Small Tips
“=” in Python means assignment, it doesn’t test equality!

Functions such as str(), int(), float() and bool() will help you convert Python values into any type.

Python List

Name a collection of values
Contain any type
Contain different types
List Slicing: [start(inclusive):end(exclusive)]

Functions
  1. max()
  2. len(): length of list or string
  3. round(number,decimal)
  4. sorted(iterable, reverse = True/False) True = descending, False = ascending
Methods

Mathods: Functions on objects (eg:list)

  1. strName.capitalize()
  2. strName.replace(“a”, “b”) replacing “a” with “b”
  3. listName.index()
  4. listName.count()
  5. lisyName.append() adding sth at the end of original list
  6. listName.reverse()

Joining and Blending Data

  1. Types of Joins
    1. Inner Join: An inner join will take the two tables and it will look at only their intersect, matching rows.
    2. Left Outer Join: Where there was a match, the rows are glued together and where there was no match the associated columns are left blank or some in SQL basically those columns will be nolde, so let’s just keep them blank for now.
    3. Right Outer Join: Compared with last Join, the second table is the main table now.
    4. Full Outer Join: Units of two tables.
  2. Blending Data
    If the data sources are different then Tableau won’t let you do it join you will have to be blending the data. There’s two reasons why you would want to do a blend, one is a little different level of granularity of the data sets and number two is the data sets are just basically different types of data sources.
  3. Two methods of Reducing duplicated data in two tables (how blending works in Tableau)
    1. Click “Data”->”Edit Relationships”->”Custom”->”Add”
    2. Change the name of “name2” in Table2 into “name1” like Table1
